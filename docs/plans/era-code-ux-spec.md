# Era Code UX Specification

## Vision

Era Code replaces the multi-terminal workflow of managing multiple OpenCode/Claude Code CLI sessions. It operates like a browser where each **tab is a project** and each project can have multiple **conversation sessions**.

- Expandable with syntax-highlighted content

## 16. Multi-Window Support (v1)

### 16.1 Behavior

- Users can open multiple Era Code windows (Cmd+N in Electron, or open app again)
- Each window is **fully independent**
- Windows do not share state or tabs
- Same project can be open in multiple windows (user's choice)

### 16.2 Limitations (v1)

- No tab dragging between windows
- No shared session state
- Opening same project in two windows = two separate OpenCode processes

### 16.3 Future (v2)

See `era-code-v2-scope.md` for:
- Linked windows with shared state
- Tab dragging between windows
- Single OpenCode process per project across windows
